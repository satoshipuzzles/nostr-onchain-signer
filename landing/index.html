How It Works

From npub to multi-sig address in 4 steps.

1

Your Nostr key IS your Bitcoin key

Every Nostr npub is a secp256k1 x-only public key — the exact same format Bitcoin Taproot uses. Your npub already has a corresponding bc1p... address. No new keys needed.

2

Select keys from your social graph

Browse your following list with profile pictures, NIP-05 verification, and display names. Search and select the npubs you want in your multi-sig. Choose any m-of-n threshold.

3

Multi-sig address derived via Tapscript

A BIP342 Tapscript is constructed using OP_CHECKSIGADD for each selected key. The script is placed in a Taproot tree with an unspendable internal key. Result: a standard bc1p... address.

4

Coordinate signing over Nostr

When you want to spend, initiate a signing round. The PSBT is passed to each signer via encrypted Nostr DMs. Track progress in real-time. Broadcast when threshold is reached.

Technical Specification

Open protocols, no proprietary magic.

OP_RETURN Protocol (NSTR)

Byte layout (39-61 bytes, max 80):
┌──────────────┬─────────┬────────┬──────────────────┬──────────────────┐
│ "NSTR" (4B)  │ Ver (1) │ Kind   │ Event ID (32B)   │ Content Hash     │
│ 0x4e535452   │  0x01   │ (2 LE) │ SHA-256 of event │ (20B, optional)  │
└──────────────┴─────────┴────────┴──────────────────┴──────────────────┘

Total sizes:
  Without content hash: 41 bytes (OP_RETURN + push + payload)
  With content hash:    61 bytes
  Maximum allowed:      80 bytes (Knots policy)

Tapscript Multi-Sig (BIP342)

Script structure for m-of-n:

  <key1> OP_CHECKSIG
  <key2> OP_CHECKSIGADD
  <key3> OP_CHECKSIGADD
  ...
  <keyN> OP_CHECKSIGADD
  <m>    OP_NUMEQUAL

Address derivation:
  1. Build Tapscript from x-only pubkeys (npubs)
  2. Compute TapLeaf hash
  3. Use unspendable NUMS internal key
  4. Q = P + tagged_hash("TapTweak", P || leaf) * G
  5. Address = bech32m("bc", 1, x(Q))

Key Derivation

npub → x-only pubkey (32 bytes)
     → BIP341 Taproot address (key-path)

Multi-sig from npubs:
  npub₁ → pubkey₁ (32B x-only)
  npub₂ → pubkey₂ (32B x-only)
  ...
  npubₙ → pubkeyₙ (32B x-only)

  script = CHECKSIGADD(pubkey₁..pubkeyₙ, threshold)
  address = P2TR(NUMS, script)

Security Model

Key Storage:
  • AES-256-GCM encryption at rest
  • PBKDF2 key derivation (600,000 iterations)
  • Auto-lock after 15 minutes
  • Keys never leave background service worker

Multi-Sig:
  • Unspendable internal key (NUMS point)
  • Script-path spending ONLY
  • m-of-n threshold enforced on-chain
  • Signing coordination via NIP-44 encrypted DMs

Extension:
  • Chrome Manifest V3 (minimal permissions)
  • Content script = message bridge only
  • No external servers required

Wallet Management

Hot wallet in the browser. Cold storage offline.

🔥 Hot Wallet (Browser Extension)

For daily signing and small amounts. Keys encrypted on-device.

  • NIP-07 signing for any Nostr web app
  • Quick Bitcoin transactions
  • Multi-sig initiation and coordination
  • Auto-lock with password protection

🧊 Cold Storage (Offline Signing)

For large amounts. Keys never touch the internet.

  • Generate keys on air-gapped device
  • Export unsigned PSBT via QR code or file
  • Sign offline, return signed PSBT
  • Hardware wallet support (Ledger/Trezor)
  • Multi-sig with offline key holders
  • Sparrow / Electrum compatible export

📱 Transaction Flow

How to send a transaction with this signer.

  • Open extension → "Send + Note"
  • Enter recipient address + amount
  • Optionally write a Nostr note to embed
  • Extension builds PSBT + signs Nostr event
  • For multi-sig: initiates signing round
  • Broadcast when all signatures collected

🔄 Multi-Sig Signing Round

Collecting signatures from your key group.

  • Create transaction → signing round starts
  • PSBT sent to co-signers via Nostr DM
  • Each signer reviews + signs in their extension
  • Progress tracked: ████░░░░ 3/7 signed
  • Threshold met → "Broadcast" button appears
  • TX broadcast to Bitcoin network

Build From Source

# Clone the repo
git clone https://github.com/nostr-onchain/signer.git
cd signer

# Install dependencies
npm install

# Development (hot-reload web view)
npm run dev

# Build extension
npm run build

# Load in Chrome:
#   1. Open chrome://extensions
#   2. Enable "Developer mode" (top right toggle)
#   3. Click "Load unpacked"
#   4. Select the dist/ folder
#   Done! Pin the extension and click to open.
